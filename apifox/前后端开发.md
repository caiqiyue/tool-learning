下面我用**“网站像餐厅”**来讲：

前端 = 餐厅大厅、菜单、点餐界面。
 后端 = 厨房、仓库、厨师、账本。
 接口 API = 服务员传话：前端说“我要用户列表”，后端返回“这是用户数据”。

## 一、你的前端技术栈分别是干什么的

### 1. Vue 3：前端页面框架

Vue 3 是用来做页面和交互的。比如登录页、表格页、弹窗、按钮点击、数据展示。

它的核心思想是：**数据变了，页面自动变。**

```
<template>
  <div>{{ username }}</div>
</template>

<script setup lang="ts">
const username = '张三'
</script>
```

可以理解为：Vue 负责“把数据变成页面”。

Vue 官方也推荐用 Vite 来创建 Vue 单页应用项目。

------

### 2. Vite：前端开发和打包工具

Vite 不是页面框架，它是“工程工具”。

它主要做三件事：

开发时启动项目：

```
npm run dev
```

打包上线：

```
npm run build
```

让开发时修改代码后页面快速刷新。

可以理解为：**Vite 是前端项目的施工队和打包机。**

------

### 3. Vue Router：前端路由

Vue Router 负责页面跳转。

比如：

```
/login        登录页
/dashboard    首页
/users        用户管理
/orders       订单管理
```

在传统网站里，访问不同 URL 可能由后端返回不同页面。

在 Vue 单页应用里，很多页面其实都是前端自己切换的，Vue Router 负责根据地址显示对应组件。

```
{
  path: '/users',
  component: () => import('@/views/UserList.vue')
}
```

可以理解为：**Vue Router 是前端的导航系统。**

------

### 4. Pinia：前端状态管理

Pinia 用来保存多个页面都要用的数据。

比如：

登录用户信息、token、权限、购物车、菜单权限、当前主题。

不用 Pinia 时，组件之间传数据会很乱。用了 Pinia，可以把公共数据放在一个“仓库”里。

```
export const useUserStore = defineStore('user', {
  state: () => ({
    token: '',
    username: ''
  })
})
```

可以理解为：**Pinia 是前端的公共数据仓库。**

Pinia 是 Vue 生态里常用的状态管理方案，经常和 Vue 3、Vue Router 一起使用。

------

### 5. Ant Design Vue：UI 组件库

Ant Design Vue 提供现成的按钮、表格、表单、弹窗、分页、菜单等组件。

比如不用自己从零写表格：

```
<a-table :columns="columns" :data-source="users" />
```

不用自己写按钮样式：

```
<a-button type="primary">提交</a-button>
```

可以理解为：**Ant Design Vue 是装修好的家具库。**

它让后台管理系统、CRM、ERP、数据平台这类项目开发更快。

------

### 6. TypeScript：给 JavaScript 加类型

JavaScript 比较自由，但大型项目里容易出错。

TypeScript 可以提前发现错误：

```
let age: number = 18
age = 'abc' // 报错
```

在前后端接口通信里，TypeScript 很重要，因为它可以约束接口返回数据格式：

```
interface User {
  id: number
  username: string
  email: string
}
```

可以理解为：**TypeScript 是代码的质检员。**

------

### 7. Axios：前端请求后端接口的工具

Axios 专门负责发 HTTP 请求。

比如前端向 Django 后端请求用户列表：

```
import axios from 'axios'

axios.get('/api/users/')
```

或者登录：

```
axios.post('/api/login/', {
  username: 'admin',
  password: '123456'
})
```

可以理解为：**Axios 是前端派出去和后端沟通的信使。**

------

## 二、这些技术怎么组合起来

一个典型流程是：

用户打开页面
 → Vue 显示页面
 → Vue Router 判断当前是哪个页面
 → 页面组件调用 Axios 请求后端
 → Django 返回 JSON 数据
 → Pinia 保存用户信息或权限数据
 → Ant Design Vue 把数据展示成表格、表单、菜单

比如用户管理页：

```
用户访问 /users
Vue Router 加载 UserList.vue
UserList.vue 用 Axios 请求 GET /api/users/
Django 返回用户列表 JSON
Vue 把数据渲染到 Ant Design Vue 表格里
```

## 三、前端和 Django 后端怎么通信

最常见方式是：**HTTP + JSON + REST API**。

### 例子：获取用户列表

前端请求：

```
GET /api/users/
```

后端返回：

```
[
  {
    "id": 1,
    "username": "admin",
    "email": "admin@example.com"
  }
]
```

前端代码：

```
const res = await axios.get('/api/users/')
users.value = res.data
```

Django 后端一般会用 Django REST Framework，也就是 DRF，来写 API。DRF 官方定位就是一个用于构建 Web API 的 Django 工具包。

------

## 四、Django 后端接口通常怎么写

后端一般有几层：

```
Model        数据库模型
Serializer   把数据库对象转成 JSON
View/ViewSet 接收请求，返回响应
URL/Router   定义接口地址
```

### 1. Model：数据库表

```
class User(models.Model):
    username = models.CharField(max_length=100)
    email = models.EmailField()
```

### 2. Serializer：转 JSON

```
class UserSerializer(serializers.ModelSerializer):
    class Meta:
        model = User
        fields = '__all__'
```

### 3. ViewSet：接口逻辑

```
class UserViewSet(viewsets.ModelViewSet):
    queryset = User.objects.all()
    serializer_class = UserSerializer
```

DRF 的 ViewSet 可以把一组相关接口逻辑放在一个类里，比如列表、详情、新增、修改、删除。

### 4. Router：生成接口地址

```
router.register('users', UserViewSet)
```

然后自动生成：

```
GET    /api/users/       获取列表
POST   /api/users/       新增
GET    /api/users/1/     获取详情
PUT    /api/users/1/     全量修改
PATCH  /api/users/1/     部分修改
DELETE /api/users/1/     删除
```

## 五、前后端通信最重要的几个概念

### 1. 请求方法

```
GET     查询
POST    新增
PUT     整体修改
PATCH   部分修改
DELETE  删除
```

### 2. 状态码

```
200 成功
201 创建成功
400 参数错误
401 未登录
403 没权限
404 找不到
500 后端报错
```

### 3. Token 登录

常见流程：

```
用户输入账号密码
前端 POST /api/login/
后端验证成功，返回 token
前端把 token 存到 Pinia/localStorage
之后每次请求都带 token
后端根据 token 判断用户是谁
```

请求头一般这样：

```
Authorization: Bearer xxxxxx
```

DRF 也有认证机制，用于把请求和用户身份凭证关联起来，再结合权限判断是否允许访问。

## 六、Axios 通常怎么封装

不要每个页面都直接写 axios，建议封装一个统一请求文件。

```
// src/utils/request.ts
import axios from 'axios'

const request = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,
  timeout: 10000
})

request.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

request.interceptors.response.use(
  response => response.data,
  error => {
    if (error.response?.status === 401) {
      // 跳转登录页
    }
    return Promise.reject(error)
  }
)

export default request
```

然后页面里这样用：

```
import request from '@/utils/request'

const users = await request.get('/users/')
```

这样做的好处是：

接口地址统一、token 统一处理、错误统一处理、超时统一处理。

## 七、前后端接口怎么约定

建议你们用一份接口文档，至少写清楚：

```
接口地址：GET /api/users/
作用：获取用户列表
请求参数：page, page_size, keyword
返回数据：id, username, email
错误情况：401 未登录，403 无权限
```

前后端最怕的问题不是技术，而是“约定不清”。

比如前端以为字段叫：

```
"userName"
```

后端返回：

```
"username"
```

页面就可能显示不出来。

## 八、推荐的接口管理、测试、监控工具

### 1. Postman：接口测试

适合手动测试接口。

你可以用它测试：

```
登录接口
查询接口
新增接口
上传文件接口
带 token 的接口
```

### 2. Apifox：接口文档 + 测试 + Mock

国内团队很常用。

它适合：

```
写接口文档
前端看接口
后端调试接口
生成 Mock 数据
团队协作
```

Mock 的意思是：后端还没写好，前端先用假数据开发。

### 3. Swagger / OpenAPI：自动接口文档

Swagger UI 可以根据 OpenAPI 规范生成可交互的接口文档，开发者可以直接在浏览器里调用接口测试。

Django 项目里常用：

```
drf-spectacular
drf-yasg
```

它们可以从 DRF 代码生成接口文档。

### 4. Insomnia：轻量接口调试

类似 Postman，适合测试 REST、GraphQL、gRPC 等接口。Insomnia 官方定位是 API 调试、设计和协作平台。

### 5. 浏览器 DevTools：前端必会工具

打开浏览器 F12，重点看 Network 面板。

你可以看到：

```
前端请求了哪个接口
请求参数是什么
请求头有没有 token
后端返回了什么
接口花了多久
状态码是多少
```

这是排查前后端问题最常用的工具。

### 6. 后端日志和监控

Django 后端建议加：

```
logging 日志
Sentry 错误监控
Prometheus + Grafana 性能监控
Nginx access log
Django Debug Toolbar 本地调试
```

## 九、一个完整登录流程

```
1. 用户在 Vue 登录页输入账号密码
2. 点击登录按钮
3. Axios 发送 POST /api/login/
4. Django 校验账号密码
5. 成功后返回 token 和用户信息
6. 前端把 token 存到 Pinia 和 localStorage
7. Vue Router 跳转到首页
8. 后续请求自动带 Authorization
9. Django 根据 token 判断用户身份
```

前端示例：

```
const res = await request.post('/login/', {
  username: form.username,
  password: form.password
})

userStore.token = res.token
localStorage.setItem('token', res.token)
```

## 十、你可以这样建立整体理解

记住这张图：

```
Vue 页面
  ↓
Axios 请求
  ↓
Django / DRF 接口
  ↓
Serializer
  ↓
Model / 数据库
  ↓
JSON 返回
  ↓
Vue 更新页面
```

一句话总结：

**Vue 负责展示和交互，Vite 负责开发构建，Vue Router 负责页面跳转，Pinia 负责公共状态，Ant Design Vue 负责 UI 组件，TypeScript 负责类型安全，Axios 负责请求后端，Django/DRF 负责提供 API。**
